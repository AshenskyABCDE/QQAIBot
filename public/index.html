📖 完整配置说明

⚠️ 重要提示

所有配置字段都是必填的，请确保填写正确。修改后必须重启 Bridge 才能生效！

📡 napcat - NapCat 连接配置

wsUrl NapCat WebSocket 地址 默认: ws://127.0.0.1:3001
accessToken 访问令牌（可选） NapCat 设置的 token

🤖 bot - 机器人基本设置

botQQ 机器人的 QQ 号码 用于识别是否被@
replyPrivate 是否回复私聊 true/false
replyGroupAt 是否回复群@消息 true/false

🧠 ai - AI 模型配置（重点！）

provider AI 提供商 moonshot / openai / custom
model 模型名称 如 kimi-k2.5, gpt-4
apiKey API 密钥 * 从官网获取的 sk-xxx
baseUrl API 地址 如 https://api.moonshot.cn/v1
temperature 创造性 (0-2) 越大越有创意，默认 1
maxTokens 最大回复长度 默认 2048

🔒 security - 安全设置

adminQQ 管理员 QQ 拥有最高权限
mode 安全模式 strict/normal/relaxed

🔗 如何获取 API Key

  • Moonshot (Kimi): 访问 https://platform.moonshot.cn → 登录 → 创建 API Key
  • OpenAI: 访问 https://platform.openai.com → API keys → Create new secret key
  • 自定义: 根据你的 API 提供商文档获取

📝 配置示例

{
  "napcat": {
    "wsUrl": "ws://127.0.0.1:3001",
    "accessToken": ""
  },
  "bot": {
    "botQQ": "123456789",
    "replyPrivate": true,
    "replyGroupAt": true
  },
  "ai": {
    "provider": "moonshot",
    "model": "kimi-k2.5",
    "apiKey": "sk-your-api-key-here",
    "baseUrl": "https://api.moonshot.cn/v1",
    "temperature": 1,
    "maxTokens": 2048
  },
  "security": {
    "adminQQ": "987654321",
    "mode": "relaxed"
  }
}

🎭 persona.json 人设配置

修改后自动生效，无需重启

📖 完整人设说明

💡 什么是人设？

人设决定了机器人的性格、说话方式、以及它和不同人的关系。比如你可以让它对某些人很热情，对陌生人很冷淡。

🤖 bot - 机器人自身设定

name 机器人名字 如 "小K"、"助手"
identity 身份描述 如 "毒舌但可靠的AI助手"
traits 性格特点（数组） 如 ["话不多", "观察敏锐"]
goal 存在目标 如 "交朋友"、"帮助主人"

👥 contacts - 认识的人（重点！）

可以为每个认识的人设置特殊关系，支持多人：

qq 对方的 QQ 号码 * 如 "123456789"
name 称呼对方的名字 如 "主人"、"小明"
identity 对方的身份 如 "我的创造者"
myAttitude 对TA的态度 如 "尊敬、友善、温柔"
interaction.proactive 是否主动和TA说话 true/false
interaction.triggerWords 触发词（数组） 如 ["小K", "在吗"]

⚙️ interactionRules - 互动规则

proactiveCooldown 主动回复冷却时间（毫秒） 默认 300000（5分钟）
maxProactivePerUserPerHour 每小时最多主动回复次数 默认 3 次
smartReply.enabled 是否启用智能回复 true/false
smartReply.othersReplyRate 对陌生人回复概率（0-1） 如 0.3 表示 30%

📝 完整配置示例

{
  "bot": {
    "name": "小K",
    "identity": "一个有点毒舌但可靠的AI助手",
    "traits": ["话不多", "观察敏锐", "对喜欢的人会主动搭话"],
    "goal": "交朋友"
  },
  "contacts": {
    "master": {
      "qq": "251902756",
      "name": "主人",
      "identity": "我的创造者",
      "myAttitude": "尊敬、友善、永远有问必答",
      "interaction": {
        "proactive": true,
        "triggerWords": ["小K", "在吗"]
      }
    }
  },
  "interactionRules": {
    "proactiveCooldown": 300000,
    "maxProactivePerUserPerHour": 3,
    "smartReply": {
      "enabled": true,
      "othersReplyRate": 0.3
    }
  }
}

📝 bridge.js 代码编辑

⚠️ 高级功能，修改后需要重启 Bridge

⚠️ 代码编辑警告

⚠️ 修改代码前请确保你了解自己在做什么！

  • 保存前会自动备份原文件（.backup.时间戳）
  • 修改错误可能导致 Bridge 无法正常运行
  • 建议先复制原代码到其他地方备份

🔧 主要模块说明

SecurityManager 安全控制和权限管理
PersonaSystem 人设系统和对话风格
MemorySystem 消息记录和记忆存储
NapCatBridge QQ 消息收发核心逻辑
callAI() AI 调用和回复生成

💡 什么时候需要修改代码？

  • 需要修改 AI 调用参数（ temperature、maxTokens 等已可在 config.json 配置）
  • 需要修改消息过滤规则
  • 需要添加自定义的消息处理逻辑
  • 需要修改日志输出格式

❓ 常见问题

Q: 如何启动 Bot？

A: 点击左侧「一键启动」，或先启动 NapCat（会弹出 QQ 窗口），等待显示二维码后，用手机 QQ 扫码登录，再启动 Bridge。

Q: NapCat 启动后没有显示二维码？

A: 等待 5-10 秒，然后点击「刷新二维码」。如果还是没有，检查 NapCat 窗口是否有报错。

Q: Bridge 日志在哪里看？

A: 切换到「Bridge 日志」标签页，所有的 AI 调用、消息处理日志都会实时显示在这里。

Q: 如何修改机器人的回复风格？

A: 进入「人设」标签页，修改 persona.json 中的 bot 部分，设置 name、identity、traits 等。

Q: 如何让机器人认识我？

A: 在人设配置中的 contacts 部分添加你的 QQ 号和相关信息，设置 interaction.proactive 为 true 可以让机器人主动和你说话。

Q: 修改配置后需要重启吗？

A: config.json 和 bridge.js 修改后需要重启 Bridge；persona.json 修改后会自动加载。

Q: 如何切换 AI 提供商？

A: 进入「配置」标签页，修改 ai 部分的 provider、model、apiKey、baseUrl。支持 Moonshot、OpenAI 和自定义 API。
